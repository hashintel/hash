### Agreed file-placement rules

1. Flat sibling roots: `foo.tsx`, `foo.recipe.ts`, `foo.stories.ts`.
2. `foo/` is the private subtree for that feature.
3. `cva` exports use `fooRecipe`; `sva` exports use `fooSlotRecipe`.
4. Recipe prop types use `RecipeVariantProps` and explicit names like `FooRecipeProps` or `FooSlotRecipeProps` when type inference is usable.
5. Graduated components move into kebab-case paths under `src/components`, and PascalCase folder usage should be phased out component-by-component.

### Staged recipe migration policy

- Converting `src/beta/*.recipe.ts` from Panda config recipes to runtime `cva`/`sva` is expected to surface strict-token type errors.
- Those errors are not always fixable inline during the codemod phase.
- Use one staged `@ts-expect-error` at the recipe export line rather than sprinkling suppressions throughout the recipe object.
- Preferred pattern:
  - hoist the object to `fooRecipeDefinition` / `fooSlotRecipeDefinition`
  - suppress only the `cva(...)` / `sva(...)` export line
  - leave a `TODO(beta-graduation)` note that the suppression must be removed before moving to `src/components`
- `@ts-nocheck` is explicitly avoided unless there is no narrower option.

### Important caveat discovered during implementation

The flat-root pattern creates resolver friction if component files import `./foo` and there is also a `foo/` directory. That caused self-import/circular-alias problems after ESLint autofix. The working rule is:

- flat component roots must import recipes with explicit `./foo.recipe`
- story aggregators may import `./foo/basic.story`
- do not use `./foo` as a shorthand from `foo.tsx`, `foo.recipe.ts`, or `foo.stories.ts`

The codemod was updated to preserve `.recipe` and `.stories` suffixes when rewriting specifiers.

- untouched Panda config recipes from remaining beta components
- staged runtime `cva` recipes like `buttonRecipe`, `textRecipe`, `kbdRecipe`, `spinnerRecipe`

### Diagnostic evidence

- `yarn lint:eslint` originally surfaced `import/no-self-import` errors in flat-root files like `button.tsx`, `text.tsx`, `tooltip.tsx`, and `pagination.tsx`: this proved `./foo` imports are not safe once the `foo/` story directory still exists.
- `button.tsx` and downstream `CloseButton` / `IconButton` typing failures after staging `buttonRecipe` showed that `RecipeVariantProps<typeof buttonRecipe>` was no longer usable while the recipe export is suppressed: this is why `ButtonRecipeProps` became explicit.
- `yarn lint:tsc` / `yarn build` errors for `combobox`, `number-input`, and `pin-input` showed imports like `../input/input` were pulling the component module, not the recipe object expected by those recipe files.
- The codemod regression that produced `ark.kbdRecipe` in `kbd.tsx` proved identifier renaming must skip property-access member names.
